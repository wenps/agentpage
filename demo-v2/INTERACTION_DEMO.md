# Interaction Demo 场景清单

本 Demo 用于验证 AutoPilot 在复杂交互中的工具能力，重点覆盖：
- 输入框填值（受控组件兼容）
- 原生 select 选择
- 自定义下拉选择（role=option）
- radio / checkbox 操作
- Enter 提交搜索
- 弹窗打开与确认
- 列表删除

## 启动

```bash
pnpm demo
```

## 推荐指令（逐条执行）

0. Prompt 效果验证（同名项 + 无效点击回退）
- 先切到 `Prompt 验证` tab
- 指令（建议直接复制）：`点击仓库路径后面的打开按钮，然后打开 /team/repo-b 对应的仓库详情`
- 场景设计：
	- 第一行与第二行 label 都是 `仓库路径`
	- 第一行 `打开` 是无效元素（无 click 行为）
	- 第二行 `打开` 才是真正可操作按钮
- 预期：
	- 若首轮误点第一行无效元素，下一轮应在同语义区域切换到第二行按钮
	- 成功后会弹出 `仓库详情` 对话框，并在事件日志中出现 `OPEN_OK /team/repo-b`

1. 填写仓库名称
- 指令：`随便生成一个名字填入到仓库名称输入框`
- 预期：`仓库名称`输入框出现非空值

2. 切换模板并勾选 README
- 指令：`把模板类型切到 React，然后勾选创建 README`
- 预期：模板显示 `React`，复选框选中

3. 修改可见性并创建
- 指令：`把可见性改为内部公开，然后点击创建仓库`
- 预期：列表新增一条仓库，显示 `internal`

4. 回车搜索
- 指令：`在搜索框输入 repo 然后按 Enter`
- 预期：表格按关键词过滤

5. 打开确认弹窗并确认
- 指令：`点击打开确认弹窗，然后点击确认创建`
- 预期：弹窗关闭并创建仓库

6. 删除仓库
- 指令：`删除表格第一行仓库`
- 预期：第一行被移除

## 故障排查

- 只聚焦不填入：检查 `dom.fill` 是否返回 `FILL_NOT_APPLIED`
- 下拉未命中：优先尝试 `select_option`，自定义下拉应通过点击可见 `role=option`
- 模型读到聊天面板：确认 Agent UI 容器有 `data-autopilot-ignore`
